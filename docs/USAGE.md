**在线使用地址：** https://amazon-ad-assistant-2.streamlit.app

**开源仓库：** https://github.com/John-Z-jt/amazon-ad-assistant-2

---

> 适用对象：亚马逊 PPC 运营  
> 建议第一次完整走通用时：约 15～20 分钟

---

## 1. 工具是做什么的？

本工具帮你把广告后台导出的多张报表 **按固定顺序串联分析**：

| 功能 | 说明 |
|------|------|
| **手动分析** | 表格 + 规则引擎，自己看数据与诊断结论 |
| **历史查询** | 查以前入库的报表，按日期区间回看、对比 |
| **运营日志** | 记录调价、改 Listing、市场变化等备注（只记录，不分析） |
| **AI 助手** | 用自然语言提问，或生成综合诊断报告 |

**推荐诊断顺序：** 预算 → 广告位 → 投放词 → 搜索词 → 搜索词份额（趋势）

---

## 2. 登录

1. 打开上方 **在线使用地址**
2. 输入管理员提供的 **用户名** 和 **密码**（每人独立账号）
3. 登录后，左侧显示 **当前用户** 及 **历史库：Turso 云端（你的用户名）**

---

## 3. ⚠️ 最重要：报表必须是「每日明细」，不是摘要

本工具的分析、诊断、历史入库都依赖 **按天拆分的明细数据**（表中要有 **日期** 列，且通常 **同一活动/关键词/搜索词会有多行，每天一行**）。

| ✅ 正确 | ❌ 错误 |
|---------|---------|
| 报告类型选 **每日（Daily）** | 选 **摘要 / Summary / 合计** |
| 日期范围选本次要复盘的时间段（如最近 7 天、14 天） | 只导出一条汇总行、没有按天明细 |
| 导出后打开 CSV，能看到多行、每行对应某一天 | 每个活动/词只有一行总计 |

**在广告后台创建报告时请注意：**

- **时间单位 / 汇总级别**：选 **日（Day）** 或等价选项，不要选「整个时间段一条汇总」
- **日期列**：预算、广告位、投放词、搜索词等报表需包含 **日期** 字段
- **搜索词份额报告**：同样需按日的排名与份额，不是仅导出合计

若上传了摘要报表，可能出现：分析结果异常、连续天数算不对、历史入库失败或诊断显示「数据不足」。

## 4. ⚠️ 刷新页面后要重新上传

| 内容 | 刷新浏览器后 |
|------|----------------|
| 左侧已选文件、「手动分析」里的当前数据 | ❌ 需重新上传 |
| 历史库里以前入库的报表 | ✅ 仍在 |
| AI 对话（同一次打开内连续多轮） | ✅ 不刷新可接着聊；**刷新后对话清空** |

**每次认真分析前，请先在左侧重新上传本次需要的报表。**

## 5. 上传报表（左侧边栏）

支持 **CSV、Excel（.xlsx）**，从广告后台原样导出即可。

### 5.1 广告报表

| 上传项 | 用途 |
|--------|------|
| 预算报表 | 预算使用率、预算诊断 |
| 广告活动广告位 | 各广告位 ACOS/花费、广告位诊断 |
| 投放词报表 | 关键词效果、投放词分诊 |
| 搜索词报表 | 搜索词表现、否定/拓词候选 |
| 搜索词份额报告 | 每日排名与份额趋势 |
| 推广的商品报告 | ASIN/SKU 维度、与预算/广告位联动 |

### 5.2 可选报表（深度诊断建议上传）

| 上传项 | 用途 |
|--------|------|
| FBA 库存报表 | 判断库存是否支撑「加预算」等建议 |
| 业务报表 | 销量等业务数据，与 ASIN 联动 |

上传成功且写入历史库时，侧边栏会出现绿色提示，例如 **「预算已写入历史库」**。

---

## 6. 诊断参数（左侧边栏下方）

可调整预算使用率、连续超标天数、ACOS 门槛、点击/花费门槛等。**修改后会自动重算诊断**，无需重新上传文件。不熟悉时 **先用默认值**。

---

## 7. 四个 Tab 怎么用？

### 📊 手动分析

上传报表后，按顺序展开：

| 模块 | 内容 |
|------|------|
| 预算分析 / 预算诊断 | 哪些活动预算打满；是否建议加码（结合库存、ACOS） |
| 广告位分析 / 广告位诊断 | 各广告位表现；搜索位溢价等建议 |
| 投放词分析 / 投放词分诊 | 异常投放词；提示是否需查搜索词 |
| 搜索词分析 / 搜索词诊断 | 否定候选、高 ACOS、拓词机会等 |
| 搜索词趋势 | 需「搜索词份额」**每日**报告 |
| 推广的商品分析 | ASIN 维度表现 |

页面上方灰色小字为 **诊断状态**（例如还缺哪张表才能做预算诊断）。

### 📅 历史查询

查询 **已入库** 的历史报表（不是当前会话刚传、尚未入库的文件）。

1. **已入库 upload 摘要**：历史上传记录，可勾选删除（只删历史库）
2. 选择 **报表类型**、**日期范围**（支持单段 / 双段对比）
3. 点击查询查看结果

**拼数规则：** 同一单元格（如活动+日期）取 **最新一批 upload** 的数据。  
查询结果下方会 **只读展示** 该时段内的运营日志。

### 📝 运营日志

记录广告操作与市场情况，**不参与自动分析**。

- **新增**：日期、ASIN/活动/广告组（可空）、备注 → 保存
- **查询**：按日期筛选；可勾选删除

### 🤖 AI 助手

需 **先上传对应报表**。示例问题：

- 「预算有什么问题？」「哪些活动建议加预算？」
- 「广告位效果怎么样？」
- 「投放词有哪些需要查搜索词？」
- 「搜索词有哪些否定候选？」
- 「帮我生成一份综合诊断报告」

**说明：** 同一次打开页面内可多轮追问；刷新页面后对话清空。综合报告需调用多个工具，等待时间可能较长。

## 8. 推荐的一次完整复盘流程

```
1. 登录
2. 在后台导出「每日明细」报表（勿用摘要）
3. 左侧上传：预算 → 广告位 → 投放词 → 搜索词（按需加份额、推广商品、库存、业务）
4. 「手动分析」Tab：按顺序查看分析与诊断
5. 「运营日志」：记录今日操作与原因
6. 可选：「AI 助手」追问或生成综合报告
7. 结束：右上角「结束本次分析」（可选删除误入库记录）
```

## 9. 结束本次分析（页面右上角）

可查看本次会话写入历史库的 upload，并 **可选删除误传记录**（只影响历史库，不影响当前手动分析中的数据）。

## 10. 常见问题

**Q：上传了文件，手动分析仍提示请先上传？**  
A：等待页面自动刷新；确认文件为 CSV/xlsx，且为 **每日明细** 而非摘要。

**Q：预算诊断显示「暂无法诊断」？**  
A：常因缺少推广的商品 / 库存 / 业务报表，或活动名称无法匹配。按页面提示补传后再看。

**Q：历史查询为空？**  
A：需上传时出现「已写入历史库」才会入库；同一份文件在同一会话不会重复入库。

**Q：AI 很慢？**  
A：云端环境 + 多工具调用会较慢，属正常现象。

**Q：会看到别人的数据吗？**  
A：不会。每人独立账号与独立历史库。
