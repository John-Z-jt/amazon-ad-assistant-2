## 优先级说明


| 级别     | 含义                  |
| ------ | ------------------- |
| **P0** | 阻塞日常使用或导致功能失败，应优先修复 |
| **P1** | 显著提升体验或补齐核心能力，近期迭代  |
| **P2** | 增强型功能或架构演进，中长期规划    |

## P0 — 必须优先

### P0-1 修复手动分析「默认无数据」体验


| 维度         | 说明                                                               |
| ---------- | ---------------------------------------------------------------- |
| **解决什么问题** | 预算、广告位、投放词等模块的 `multiselect(default=[])` 导致用户上传报表后看不到汇总表，误以为分析失败 |
| **开发难度**   | ⭐ 低（1–2 天）：调整默认选中逻辑或上传后直接展示全量汇总                                  |
| **用户价值**   | ⭐⭐⭐⭐⭐ 极高：手动分析 Tab 是核心功能，首屏体验直接影响可用性                              |
| **是否推荐开发** | ✅ **强烈推荐**，投入产出比最高                                               |
